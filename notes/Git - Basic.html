Basic Git Commands

git add

  • adds files to the staging area
git add <file_name>

git commit

  • records changes in the repository
git commit -m "commit message"

git push

  • uploads local repository content to a remote repository.
git push

git clone

git clone <repository_url>

git pull

  • Fetches from and integrates with another repository or a local branch
git pull

git branch

  • lists / creates / deletes branches
git branch # list all branches
git branch <branch_name> # create new branch
git branch -d <branch_name> # delete branch

Undoing Changes

  • Working directory: It contains the current state of the project, including any changes that have not yet been committed
  • Staging area: You add changes to the staging area (using git add) to tell Git what you want to include in the next commit
  • HEAD usually pointing to the latest commit on the current branch

git reset

  • Reset current branch to a specified state
  • Options:
    • --soft Keeps staging area and working directory unchanged
    • --hard Resets staging area and working directory, discarding all changes.

  • Usage:

    • change the current working directory to the commit one before the latest commit

      bash git reset --hard HEAD~1

    • Revert the commit

      • This command creates a new commit that undoes the changes made by the last commit. It does not rewrite history, which is safer for shared repositories

        bash git revert HEAD git push

    • Rewriting history (Use with Caution)

      • Use with caution, only if remote changes are not pulled by others.

      • move the branch back to the previous commit and overwrite the history on the remote repository

        bash git reset --hard HEAD~1 git push --force

    • Undo last commit

      bash git reset --soft HEAD~1

git config --global user.email "you@example.com"
